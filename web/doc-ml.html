2. As 7 Classes Compostas

Cada classe combina velocidade do motor com estado do eixo (girando ou parado). Isso permite distinguir, por exemplo, um motor configurado como HIGH mas cujo eixo parou de girar (HIGH_ROT_OFF) de um que esta genuinamente desligado (FAN_OFF).

Label Descricao Condicao Fisica
LOW_ROT_ON Velocidade baixa — eixo girando Motor em LOW, helice em rotacao
MEDIUM_ROT_ON Velocidade media — eixo girando Motor em MEDIUM, helice em rotacao
HIGH_ROT_ON Velocidade alta — eixo girando Motor em HIGH, helice em rotacao
LOW_ROT_OFF Velocidade baixa — eixo parado Motor em LOW, helice parada (preso/desequilibrado)
MEDIUM_ROT_OFF Velocidade media — eixo parado Motor em MEDIUM, helice parada
HIGH_ROT_OFF Velocidade alta — eixo parado Motor em HIGH, helice parada
FAN_OFF Ventilador desligado Sem alimentacao, sensor em repouso
Atencao: Essas 7 classes de ML sao diferentes dos 4 modos de controle da API (LOW / MEDIUM / HIGH / PAUSE). O modelo infere o estado real do hardware, que pode diferir do modo comandado.

3. Feature Engineering

A partir de cada janela deslizante de 100 amostras (5 segundos a 20 Hz, passo de 1 segundo), sao calculadas 13 metricas por eixo, totalizando 104 features candidatas.

8 Eixos Sensoriais
  • accel_x_g — aceleracao X (g)
  • accel_y_g — aceleracao Y (g)
  • accel_z_g — aceleracao Z (g)
  • gyro_x_dps — giroscopio X (°/s)
  • gyro_y_dps — giroscopio Y (°/s)
  • gyro_z_dps — giroscopio Z (°/s)
  • vibration_dps — magnitude giroscopio (°/s)
  • accel_mag_g — magnitude aceleracao (g)

vibration_dps = √(gx² + gy² + gz²) — derivado.
accel_mag_g = √(ax² + ay² + az²) — derivado.

13 Metricas por Eixo
  • Temporais: std, range, rms
  • Forma: skew, kurtosis
  • Percentis: p10, p25, p75, p90, p95
  • FFT bandas:
    • fft_low — 0–5 Hz
    • fft_mid — 5–10 Hz
    • fft_high — 10–20 Hz (Nyquist)
Janela deslizante: 100 amostras × 50 ms = 5 segundos de contexto por predicao. Passo de 20 amostras (1 segundo) gera sobreposicao de 80% para maximizar o dataset com 2.255 janelas. A funcao extract_features_windowed_extended() implementa esse processo no notebook 02_Feature_Engineering.ipynb.

As bandas FFT sao calculadas via RMS da amplitude discreta (DFT) dentro de cada faixa de frequencia. No browser, Stats.fftBandRms(arr, fLow, fHigh, 20.0) realiza DFT real em O(n²) sobre janelas de n=100 pontos — viavel para inferencia em tempo real.

4. Selecao de Features — Cohen's d + TOP-K

A selecao usa Cohen's d minimo entre todos os 21 pares de classes (d_min_all) como criterio de separabilidade, combinado com filtro de correlacao e selecao TOP-K por importancia de floresta.

Criterio de Selecao

min_cohens_d 0.30 — limiar de separabilidade minima
score_formula d_min_all — pior par entre as 7 classes
correlation_threshold 0.85 — remove features redundantes
top_k 16 — features finais selecionadas
Candidatas avaliadas 104 (8 eixos × 13 metricas)
Aprovam d ≥ 0.30 ~14 features

Interpretacao do Cohen's d

dSeparabilidade
< 0.20Insignificante
0.20 – 0.49Pequena
0.50 – 0.79Media
≥ 0.80Grande
O par mais dificil do dataset e LOW_ROT_ON ↔ LOW_ROT_OFF para features como vibration_dps_p10 (d=0.427). O RF supera o GNB precisamente nesses pares limítrofes.

Cada arvore e serializada como arrays planos (flat arrays) compatíveis com traversal eficiente em JavaScript:

// Estrutura por arvore { "feature": [0, 3, -2, ...], "threshold": [1.23, 0.45, -2.0, ...], "children_left": [1, 3, -1, ...], "children_right": [2, 4, -1, ...], "value": [[0,0,10,...], ...] } // feature[i] == -2 indica no-folha // children == -1 indica ausencia de filho

6. Inferencia em Tempo Real — classifier.js v6.0

O arquivo web/js/classifier.js implementa toda a inferencia no browser, sem dependencias externas. Funciona com tres tipos de modelo: Random Forest, Softmax LogReg e Gaussian NB.

Stats.percentile()
  • Interpolacao linear alinhada com np.percentile
  • Calcula P10, P25, P75, P90, P95 por janela
  • Ordena o buffer in-place para eficiencia
Stats.fftBandRms()
  • DFT real discreta, O(n²), n=100
  • Parametros: fLow, fHigh, samplingHz=20.0
  • Retorna RMS das amplitudes na banda
  • Bandas: 0–5 Hz, 5–10 Hz, 10–20 Hz
RandomForestClassifier._predictTree()
  • Traversal de arvore via arrays planos
  • Loop while: vai para filho esq/dir pelo threshold
  • No-folha identificado por feature==-2
  • Voto final: media das distribuicoes de probabilidade

Fluxo de Inferencia em Tempo Real

ClassifierConfig.SAMPLING_HZ 20.0 Hz (deve casar com treinamento)
WINDOW_SIZE 100 amostras (5 segundos)
MIN_POINTS 40 (2 segundos para primeira predicao)
HYSTERESIS_COUNT 3 predicoes consecutivas para mudar classe
PREDICTION_INTERVAL_MS 250 ms (4 predicoes/segundo)
Dispatch por tipo random_forest → RF  |  softmax_logreg → LogReg  |  default → GNB

Pipeline por ciclo de 250 ms:

  1. Buffer circular recebe novos pontos do sensor (via WebSocket/polling)
  2. FeatureExtractor.extract(buffer) calcula as 16 features
  3. RealTimeClassifier.predict(features) dispara o modelo
  4. Histerese: acumula predicao; muda estado so apos N confirmacoes
  5. Deteccao de mudanca brusca: flush parcial do buffer ao detectar transicao
  6. Resultado emitido via callback para o dashboard

8. Rastreabilidade

O projeto mantem tres arquivos de rastreabilidade para garantir reproducibilidade completa do pipeline de ML.

feature_config.json
  • Versao: 5.16
  • Hash SHA-256 do CSV de features usado no treino
  • 16 features selecionadas e 104 candidatas
  • Criterios de selecao completos
  • Ranking Cohen's d de todos os pares
  • Distribuicao de classes e collection IDs
pipeline_registry.json
  • Historico de runs de Feature Engineering
  • Historico de runs de treinamento
  • Hashes SHA-256 de inputs e outputs
  • Associa cada modelo ao CSV que o gerou
  • Permite auditoria completa da linhagem de dados
MODEL_REGISTRY.json
  • Historico de todos os modelos exportados
  • Metadados: versao, data, notebook, features
  • Status: deployed / archived / failed
  • Referencia ao modelo pai (parent_model)
  • Acuracia CV e configuracao de feature_config
