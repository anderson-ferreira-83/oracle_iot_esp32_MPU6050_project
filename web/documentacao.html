Rastreabilidade (Sessão, Taxa e Rótulos)
  • collection_id (sessão): identifica o lote atual de coleta. Deve refletir a taxa (ex.: _15hz).
  • sample_rate: taxa configurada no Controle e aplicada ao pipeline.
  • mode: rótulo operacional (RAW/LOW/MEDIUM/HIGH/PAUSE) usado para organizar dataset e inferência.
  • ingest (DB ON/OFF): quando OFF, os dados podem continuar chegando, mas não são gravados no Oracle XE.
Muito importante: se collection_id indicar 5Hz e a taxa estiver em 15Hz, você corre o risco de contaminar o dataset e gerar logs inconsistentes. Por isso o sistema mostra alertas e oferece criar uma nova sessão.
Controle (control.html): O Que Cada Etapa Faz
Etapa 1: Preparação
  • Nova Coleta: cria uma nova sessão (collection_id).
  • Taxa: ajusta sample_rate (ex.: 15Hz).
  • Pausar DB / Retomar DB: liga/desliga a gravação no Oracle XE (ingest_enabled).
  • Zerar Banco: apaga as leituras do Oracle XE para reiniciar um experimento.
  • Mismatch: alerta quando o nome da sessão não combina com a taxa.
Ver documentação detalhada →
Gestão de Rede (Hotspot + Doméstica)
  • Perfis Wi-Fi: define SSID, senha e IP do servidor por rede.
  • Aplicar no ESP32: envia configuração via próximo POST; o dispositivo reinicia.
  • Abrir Portal: solicita ao ESP32 que abra o portal de configuração Wi-Fi.
  • Diagnóstico: mostra status de WiFi, servidor e fluxo de dados.
Ver documentação detalhada →
Etapa 2: Validação da Taxa
  • Coloca o sistema em RAW e mede a taxa real.
  • Exibe jitter, intervalo médio e eficiência.
  • Serve para confirmar que 15Hz realmente está chegando como 15Hz.

  • Exibe estatísticas da coleta e últimos envios.

Dashboard (index.html): O Que Cada Card Faz

A dashboard é o “painel executivo” do sistema. Ela consome os dados mais recentes (e histórico) e exibe classificação, gráficos e testes para rastrear estabilidade e transições.

Classificação ML (unificada)
  • Mostra classe atual, confiança, buffer e estabilidade.
  • Botão Detalhes revela debug (bruto vs suavizado, histerese, features).
  • Botão Teste exibe/oculta a seção de testes guiados.
Testes (Transição + Soak)
  • Transição: passos guiados com tempos de início/fim e marcação de forçados.
  • Soak: coleta por estado com espera máxima e logs detalhados.
  • Os testes geram logs para análise e para o fluxo de aprendizado online.
Desempenho do Modelo + Aprendizado Online
  • Exibe métricas do modelo (CV, Train, Holdout, confusão L↔M).
  • Aprendizado Online fica em <details> para reduzir altura do card.
  • Atalho aponta para a absorção da sessão de Soak (quando existir).
Gráficos e Controles
  • Abas por tipo: Gyro detalhado, Accel detalhado, Visão geral.
  • Janela de tempo + Zoom X (X-/X+) controlam o eixo X.
  • Zoom Y (Y-/Auto/Y+) ajusta a leitura do eixo Y por gráfico.
Sobre “Auto”: o Auto ajusta limites dinamicamente. Já o Zoom Y define um “range mínimo” para não colapsar o gráfico quando o sinal fica muito estável.
Modelo de ML — Random Forest 7 Classes

O classificador usa um Random Forest com 200 arvores treinado sobre 16 features selecionadas de 104 candidatas (FFT + percentis + estatísticas temporais), atingindo 97.34% de acuracia CV em 7 classes compostas de velocidade × rotacao.

7 Classes de ML
  • *_ROT_ON: LOW / MEDIUM / HIGH — eixo girando
  • *_ROT_OFF: LOW / MEDIUM / HIGH — eixo parado
  • FAN_OFF: ventilador desligado
Pipeline de Features
  • 8 eixos × 13 métricas = 104 candidatas
  • Filtro Cohen’s d ≥ 0.30 + correlacao + TOP-16
  • Janela: 100 pts (5 s a 20 Hz), step=20 (1 s)
  • FFT: 0–5 Hz, 5–10 Hz, Percentis P10–P95
Modelos Exportados
  • rf_model_20260222.json — 97.34% CV (primário)
  • gnb_model_20260222.json — 93.70% CV (fallback GNB)
  • Inferência no browser via classifier.js v6.0

Logs e Arquivos Importantes
  • api/get_data: leitura (latest/histórico) usada pelo dashboard.
  • api/ingest: ingestão no Oracle XE (respeita DB ON/OFF).
  • api/set_mode: modo, taxa e sessão (usado pelo Controle).
  • api/log_transition: logs dos testes (Transição/Soak), com tempos e resumo.
  • models/: arquivos JSON do modelo e índice por taxa.
FAQ (Problemas Comuns)
  • Alerta de mismatch (5Hz vs 15Hz): crie nova sessão com o botão sugerido no Controle.
  • DB OFF: útil para pausar gravação sem parar a UI, mas estatísticas baseadas no DB ficam congeladas.
  • Offline na dashboard: verifique Wi‑Fi/ESP32 e se o servidor backend FastAPI está rodando.
  • Zoom estranho: use Auto (Y) ou resetando o dashboard para limpar cache local.
