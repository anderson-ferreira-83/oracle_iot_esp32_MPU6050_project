Etapa 4: Timer de Coleta

Timer visual para padronizar a duracao de cada rotulo. Garante consistencia entre sessoes e facilita a comparabilidade do dataset.

Timer Consistencia Padronizacao
Abrir Controle Visao Geral
Objetivo Padronizar o tempo de coleta por classe para datasets equilibrados.
Formato Display MM:SS com meta e indicador visual de progresso.
Integracao Controlado automaticamente pelo modo Automacao da Etapa 3.
Display do Timer Controles Integracao com Etapa 3 Link para Dashboard
Display do Timer (MM:SS)

O timer exibe o tempo decorrido no formato MM:SS com indicadores visuais de progresso.

Elementos Visuais
  • Tempo decorrido: contagem progressiva MM:SS
  • Meta: tempo alvo selecionado na Etapa 3
  • Cor verde: timer em andamento, dentro do limite
  • Cor vermelha: tempo excedido (passou da meta)
Comportamento
  • Conta progressivamente (0:00 → meta)
  • Ao atingir a meta, muda para vermelho
  • No modo automatico, a classe troca ao atingir a meta
  • No modo manual, o timer continua mas indica excesso
Botoes de Controle
Iniciar
Iniciar: comeca a contagem do timer. Normalmente ativado automaticamente ao selecionar uma classe na Etapa 3.
Pausar
Pausar: congela a contagem. Util se voce precisa ajustar algo sem perder o progresso do timer.
Zerar
Zerar: reseta o timer para 0:00. Use ao iniciar a coleta de uma nova classe.
Interacao com Etapa 3: ao clicar em LOW, MEDIUM ou HIGH na Etapa 3, o timer e automaticamente zerado e iniciado. Ao clicar PAUSAR, o timer tambem pausa.
Integracao com Automacao

Quando o modo automatico da Etapa 3 esta ativo, o timer e totalmente controlado pela automacao.

  • A automacao inicia o timer ao entrar em cada classe.
  • Ao atingir a meta, a automacao troca para a proxima classe (com transiente).
  • O timer e zerado automaticamente a cada troca de classe.
  • Os botoes manuais (Iniciar/Pausar/Zerar) ficam desabilitados durante a automacao.
  • Ao cancelar a automacao, os botoes voltam a funcionar normalmente.
Dica: no modo automatico, o timer serve apenas como indicador visual — voce nao precisa interagir com ele. Toda a logica de tempo e gerenciada pela automacao da Etapa 3.
Proximo Passo: Dashboard

Apos concluir a coleta de treinamento, o proximo passo e abrir o Dashboard para validar os dados e testar o classificador em tempo real.

O que fazer no Dashboard:
  1. Verificar graficos: confirme que as leituras dos sensores fazem sentido para cada classe.
  2. Testar classificacao: o classificador ML deve identificar corretamente a classe atual.
  3. Executar Transition Test: teste as transicoes entre classes (LOW→MED→HIGH).
  4. Executar Soak Test: teste a estabilidade de cada classe por tempo prolongado.
  5. Absorver dados (Online Learning): se necessario, use os dados do Soak Test para refinar o modelo.
