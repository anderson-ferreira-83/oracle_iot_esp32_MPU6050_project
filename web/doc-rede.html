Banner de Status do ESP32

No topo da secao de rede, um banner mostra o estado atual do dispositivo, atualizado a cada 10 segundos via fetchDeviceStatus().

EstadoCorSignificado
OnlineVerdeESP32 enviou dados nos ultimos 30s. Exibe IP e SSID.
StaleAmareloUltimo dado entre 30s-120s atras. Pode ser reinicializacao.
OfflineVermelhoSem dados ha mais de 120s. Verifique conexao.
  • IP: endereco IP do ESP32 na rede atual.
  • SSID: nome da rede Wi-Fi a qual o ESP32 esta conectado.
  • Esses dados sao enviados pelo ESP32 junto com os payloads de sensor (diagnostico de rede).
Cards de Perfis Wi-Fi

Cada perfil de rede e exibido como um card editavel. O sistema suporta multiplos perfis com prioridade definida pela ordem (drag-and-drop para reordenar).

Campos por Perfil
  • SSID: nome da rede Wi-Fi
  • Senha: senha da rede (oculta por padrao)
  • Alias: nome amigavel para identificacao
  • Server IP: IP do servidor para esta rede (opcional)
Prioridade e Reordenacao
  • O perfil no topo tem prioridade mais alta
  • Arraste os cards para reordenar
  • O ESP32 tenta conectar na ordem definida
  • Se a rede #1 falhar, tenta #2, #3...
Server IP por perfil: cada rede pode ter um IP de servidor diferente. Util quando o servidor FastAPI tem IPs distintos em redes diferentes (ex.: 192.168.1.100 em casa, 192.168.4.1 no hotspot). Se nao informado, usa o IP global.
  • Clique no icone de lixeira para remover um perfil.
  • Use o botao "Adicionar Perfil" para criar um novo.
  • Perfis sao salvos no servidor como wifi_profiles.json.
  • O ESP32 recebe os perfis via OTA na proxima comunicacao.
Servidor & API Path

Configuracao do hostname e caminho da API usados pelo ESP32 para enviar dados.

Server Host
  • Hostname ou IP do servidor FastAPI
  • Suporta .local (mDNS) ou IP direto
  • Ex.: meupc.local ou 192.168.1.100
API Path
  • Caminho relativo da API no servidor
  • Padrao: /api
  • O ESP32 concatena host + path para montar a URL
Resolucao de IP: o ESP32 tenta, em ordem: (1) server_ip do perfil Wi-Fi conectado, (2) IP global configurado, (3) hostname .local via mDNS. Se nenhum funcionar, o dado nao e enviado.
Botoes de Acao

Tres botoes controlam a aplicacao de configuracao de rede no ESP32.

Aplicar no ESP32
Aplicar no ESP32: salva os perfis de rede e configuracao do servidor no estado pendente. Na proxima comunicacao HTTP do ESP32, ele recebe a nova configuracao, salva em flash (wifi_profiles.json + device_config.json) e reinicia automaticamente.
Cancelar Pendencia
Cancelar Pendencia: remove a configuracao pendente antes que o ESP32 a receba. Util se voce mudou de ideia apos clicar "Aplicar" mas antes do dispositivo comunicar.
Abrir Portal no ESP32
Abrir Portal: envia um comando remoto para o ESP32 abrir o portal de configuracao Wi-Fi (hotspot + pagina web). O ESP32 inicia o boot_portal.py e reinicia apos o usuario configurar. Pede confirmacao antes de enviar.
Fluxo OTA: Controle salva config no servidor → ESP32 faz POST → resposta inclui target_network_config → ESP32 processa, salva em flash e reinicia. O flag e auto-limpo no servidor apos o envio, garantindo que nao seja reenviado.

Tres indicadores visuais resumem a saude da cadeia de comunicacao entre ESP32, rede e servidor.

Dica: se Wi-Fi esta OK mas Servidor esta em falha, verifique se o backend FastAPI esta rodando e se o IP/hostname esta correto na configuracao.
Fluxos Tipicos
Primeira vez (rede nova)
  • Adicione um perfil com SSID + senha
  • Configure o Server IP para esta rede
  • Clique "Aplicar no ESP32"
  • Aguarde o ESP32 reiniciar e conectar
  • Confira o banner: Online + IP correto
Mudanca de rede (ex.: casa → lab)
  • Reordene os perfis (rede desejada no topo)
  • Verifique o Server IP do perfil de destino
  • Clique "Aplicar no ESP32"
  • O ESP32 reinicia e tenta a nova rede
  • Se falhar, tenta as redes seguintes na lista
Portal de emergencia: se o ESP32 nao conseguir conectar a nenhuma rede, ele abre automaticamente o portal Wi-Fi (hotspot). Voce pode forcar isso pelo botao "Abrir Portal no ESP32".
